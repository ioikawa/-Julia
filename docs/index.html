はじめに

Julia言語とは

Julia言語 はひと言で言えば，数値計算向けに設計された汎用プログラミング言語 である． 2018年にバージョン 1.0 がリリースされ[1]，言語仕様が正式なものとなった． 数値計算のプログラムを「簡潔に記述しやすく，実行速度も速い」というのが 最大のメリットである．

数値計算のプログラムは古くからFortranやC/C++言語で書かれることが多かったが， 最近ではMATLABやPython(+Numpy&SciPyライブラリ)などの記述しやすい言語を， 特に数値計算の講義や理論的な研究用途で，使う人が増えている． しかし，MATLABやPythonは実行速度が遅いという言語設計上避けようのないデメリットがある． Julia言語ではJITコンパイルなどの現代的な技術を駆使して，記述の簡潔性と 実行速度を高度なレベルで両立させることに成功している．

特徴

  • 文法・構文に関してはMATLABから大きな影響を受けている．
  • MATLABと同様に数学関数，有理数，複素数，任意精度計算，線形代数，などの数学に関する多数の機能が標準で実装されている．
  • MATLABはプロプライエタリ（ソースコード非公開）であるが，Julia言語はオープンソース
  • MATLABやPythonはインタプリタ型（＋動的型付け）なのでコードを簡潔に書きやすいが，その代償として実行速度が遅くなる． Julia言語はJITコンパイル等により実行速度の損失を小さく抑えている．実際に，最速のC言語と比べてもおよそ数分の1と評価されている．[2]
Note

Julia言語は数値計算に最適なプログラミング言語であるが， 数値計算に特化・先鋭化されているわけではないし， 他の言語ではできないことを可能にするような魔法の言語ではない． これは裏を返せば，Julia言語に関する知識の大部分は普遍的で他言語で通用するということでもある．

Note

Juila言語ではプログラムの初回実行時にJITコンパイルが行われる．初回は（JITコンパイル）＋（プログラムの実行時間）がトータルの実行時間になるので，遅く感じるかもしれない． 2回目以降はJITコンパイルは行われないので，プログラム本来の実行時間になる．

実行環境

本講義の基準環境

全学計算機システムではJuliaがインストール済みである． 本講義では次の環境を想定する．

  1. UbuntuサーバーにSSH接続
  2. Windows/Ubuntuのデスクトップ端末 or リモートデスクトップ
Note

もちろん，個人のPCにJuliaをインストール してレポート作成に使用してもよい． ただし，インストールや実行時のエラーについては環境に依存するので本講義ではサポートできない． Jupyter Notebookというウェブブラウザ上で Julia を実行するソフトウェアもある （インストール方法）．

Juliaの起動

sshによるリモートログイン

全学計算機システムの Ubuntuサーバに sshでログインし，Juliaを起動する手順を説明する．

macOSやUbuntu，あるいは Windows Subsystem for Linuxのターミナルから，ssh コマンドでリモートログインする．

$ ssh (全学計算機システムのユーザー名)@ubuntu.u.tsukuba.ac.jp
# パスワードを要求されるので入力

全学計算機のUbuntuサーバアドレスは「サーバー情報（学内限定）」 に書いてある．

ターミナルを起動し，julia と入力して実行するとJuliaが起動し，次のように表示される．

$ julia
               _
   _       _ _(_)_     |  Documentation: https://docs.julialang.org
  (_)     | (_) (_)    |
   _ _   _| |_  __ _   |  Type "?" for help, "]?" for Pkg help.
  | | | | | | |/ _` |  |
  | | |_| | | | (_| |  |  Version 1.7.2 (2022-02-06)
 _/ |\__'_|_|_|\__'_|  |  https://github.com/conda-forge/julia-feedstock
|__/                   |

julia>

これは REPL[repl] モードと呼ばれる状態で， プログラムの入力・実行を対話的に行うことができる． プロンプト julia> のあとにプログラムを打ち込んで実行すると， 結果がその下に表示される．

デスクトップ環境

全学計算機システムのWindows端末（あるいはリモートデスクトップ） で実行する方法について説明する．Ubuntuでもほとんど同じ手順であると思う．

Windows端末にログイン後，Windowsメニューから Juliaのアイコンを見つけて起動する． あるいは，検索窓から juliaとタイプした方が早いかもしれない．

起動したら，まず，カレントディレクトリを確認する．

julia> pwd()

ここでユーザーフォルダー C:¥¥Users¥¥〈ユーザー名〉 と表示されれば問題ないが， 私がリモートデスクトップで試した限りでは， C:¥¥Users¥¥Administrator と表示されるので注意が必要である．

フォルダー移動は cd() (change director) コマンドで行う．cd()は引数無しで実行すると ユーザーフォルダーに移動する．

julia> cd()

このあと，pwd()でカレントディレクトリを再度確認．

julia> pwd()

C:¥¥Users¥¥〈ユーザー名〉が出力されれば大丈夫である． もちろん，cd("C:¥¥Users¥〈ユーザー名〉")とベタ打ちしてもよい．

次に適当なテキストエディタ，例えば Visual Studio Codeなどで， 次の内容でhellow.jlという名前で保存する．

println("Hello, world")

Juliaのターミナルに移動して

julia> include("hellow.jl")

を実行すると，Hello, world と表示されるはずである．

Jupyter Notebook (本講義では非サポート)

Windowsメニューから Jupyter Notebookを起動する． ブラウザが立ち上がり，カレントフォルダのファイルリストが表示される． 左上側にある Newのプルダウンメニューから Julia 1.7.2 を選択する． そうすると，Juliaカーネルが起動した状態で空の "ノートブック" が表示される．

In [ ]: の右側に versioninfo() と入力して Shift+Enterを押下するとコマンドが実行される． 以下のように表示されていれば正常と思われる．

Commit bf53498635 (2022-02-06 15:21 UTC)
Platform Info:

  CPU: Intel(R) Xeon(R) CPU E5-2699 v4 @ 2.20GHz

  LIBM: libopenlibm

プログラムと実行結果がひとつのファイルに収まるし， MarkdownでLaTeXによる数式も記述できるので便利である． レポート提出用のpdfファイルも，メニューFile> Download as > HTML (.html) で変換したhtmlファイルを 印刷でpdfに変換すれば簡単に生成できる．

しかし，現時点では安定した環境であると言い切れないので，本講義ではサポートできない．
