型の変換と昇格

型の変換: convert

変数の型の変換は convertで行う．

julia> x = 100100
julia> @show typeof(x)typeof(x) = Int64
Int64
julia> y = convert(UInt32, x)   # 符号なし32bit整数へ変換0x00000064
julia> @show typeof(y)typeof(y) = UInt32
UInt32

上記のコードであれば，UInt32(x)でも同じ結果を与える． コーディングスタイルとしてどちらがよいのかはわからない．

異なる型どうしの変換が定義されていない場合はエラーとなる．

julia> y = convert(UInt32,-100)   # エラー！ERROR: InexactError: trunc(UInt32, -100)

もちろん，文字列から整数型への変換も未定義でエラーとなる．

julia> convert(Int64, "123")   # エラー！ERROR: MethodError: Cannot `convert` an object of type String to an object of type Int64

Closest candidates are:
  convert(::Type{T}, !Matched::ColorTypes.Gray24) where T<:Real

このような場合は parse関数を用いる．

julia> parse(Int64, "123")123

型の昇格: promote

promote関数は与えられた複数の変数に対して， 値の情報を損なわないように型を揃えてくれる． これを昇格 (promotion) とよぶ．

julia> promote(100, 1.5)(100.0, 1.5)
julia> typeof(ans)Tuple{Float64, Float64}

型によっては昇格できないこともある．

julia> promote(100, "abc")ERROR: promotion of types Int64 and String failed to change any arguments

異なる数値型どうしの演算では自動的にpromoteが呼び出されて適切な型変換が行われるので， 大抵の場合，型の変換を意識する必要はない．

julia> x = Int64(100)100
julia> y = Float64(0.1)0.1
julia> typeof(x+y)Float64
